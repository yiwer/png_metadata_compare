# PNG 元数据对比工具 — 舒适化重设计 + 性能专项 (Design Spec)

日期：2026-06-12
范围：前端 UI/UX 重构（渐进式改造路线）+ 两项后端配合（扫描取消、目录选择卡顿修复）。
前序：本设计在 2026-05-05 重设计的基础上演进，字段中文映射（labels.ts）、值规则、diff 引擎、扫描/配对逻辑全部沿用不动。

## 0 · 背景与目标

用户反馈的三个不舒适点 + 一个性能问题：

1. **操作路径**：看完一对要「返回目录 → 找位置 → 点下一对」，来回跳令人烦躁
2. **展示方式**：纯黑底 + 高饱和霓虹色 + 满屏徽章，长时间排查疲劳
3. **空间利用**：左右双 pane 镜像 + 底部差异汇总条互相挤压
4. **性能**：点「浏览」打开系统目录选择器时明显卡顿（目录约 600 个 PNG）；目录扫描卡死已由「命令异步化 + 进度 Channel」解决

新需求：**目录比对结果增加搜索框**（只搜文件名）。

### 已确认的设计决定

| 决策点 | 选择 |
|---|---|
| 布局骨架 | 主从布局：左侧常驻列表栏 + 中央详情（淘汰整页切换） |
| 视觉基调 | 柔和深色（石墨底、降饱和差异色），淘汰纯黑高对比 |
| 对比树形态 | 单树双值列（字段 \| 左值 \| 右值），淘汰双 pane 镜像 |
| 差异汇总 | 右侧常驻差异栏（可收起），淘汰底部条 |
| 搜索范围 | 仅文件名（纯前端过滤，后端不动） |
| 实施路线 | 甲 · 渐进式改造（在现有组件上定向手术，每步可运行） |

## 1 · 信息架构与布局骨架

### 顶栏合并为一行

「顶栏 + 槽位条 + 控件条」三条横杠压缩为一条顶栏（高约 36px，保留 `data-tauri-drag-region`）：

```
[图标] PNG Compare   [单文件|目录]              [─][□][✕]
```

- 目录路径 → 紧凑芯片移入侧栏顶部（hover 浮层显示完整路径，点击替换）
- 视图切换 / 仅看不同 / 高亮 → 移入详情区头部
- 竖向净省约 70px

### 三栏骨架（目录模式）

```
┌────────────────────────────────────────────────────┐
│ 顶栏（一行）                                         │
├──────────┬───────────────────────────┬─────────────┤
│ 侧栏 260px│ 中央详情（自适应）           │ 差异栏 220px │
│ 目录芯片  │ 详情头部：文件名+视图切换+开关 │ 差异计数     │
│ 搜索框    │ 单树双值列 / JSON / 图片     │ 分组差异清单 │
│ 筛选芯片  │                           │ 复制差异清单 │
│ 结果列表  │                           │             │
│ 底部统计  │                           │             │
└──────────┴───────────────────────────┴─────────────┘
```

- 侧栏与差异栏都可一键收起（`Ctrl+B` / `Ctrl+Shift+B`），收起状态会话内记忆，不持久化
- 窗口宽度 < 1000px 时差异栏自动收起，恢复 ≥ 1000px 时自动展开；用户手动操作过后，本会话内以手动状态为准
- 单文件模式：无侧栏；中央 + 差异栏照常；左右文件名芯片显示在详情头部（hover 全路径、点击替换）

### 四种状态

| 状态 | 表现 |
|---|---|
| 欢迎态（无输入） | 中央两个大拖放槽 + 最近目录 MRU 列表 + 快捷键提示 |
| 扫描中 | 侧栏列表区显示进度条 + 「已比对 n / total」+ 取消按钮 |
| 巡检态 | 列表选中谁，中央与差异栏即时显示谁 |
| 单边态（solo） | 只填一边或列表中的仅左/仅右项：单值列树，无差异色，差异栏隐藏 |

### 目录列表项 → 详情的映射

| 列表项状态 | 中央显示 |
|---|---|
| 不一致 / 一致 | 单树双值列（一致项无差异色） |
| 仅左 / 仅右 | solo 单值列树 |
| 错误 | 错误详情卡（错误码 + 路径 + 「打开文件」按钮），不冒充对比视图 |

## 2 · 交互细节

### 侧栏列表

- **搜索框**：输入即过滤，文件名子串匹配、不分大小写；与筛选芯片取交集；焦点在搜索框时 `Esc` 清空；匹配数实时反映在底部统计
- **筛选芯片**：`不一致 / 仅左 / 仅右 / 一致 / 错误 / 全部`，常驻计数；扫描完成默认选「不一致」，若不一致为 0 则落在「全部」且列表区显示「两侧完全一致」状态条
- **排序**：差异数降序（默认）/ 文件名 A-Z，列表头右侧小图标切换。差异数降序在混合状态（筛选=全部）下的次序：不一致（按差异数降序）→ 仅左 → 仅右 → 错误 → 一致，同组内按文件名 A-Z
- **列表行**：状态点 + 文件名（截断，hover 浮层显示左右完整路径）+ 差异数徽章（仅左/仅右项显示文字徽章）；右键菜单：复制路径 / 在资源管理器中显示
- **空态**：搜索无结果时显示「无匹配 · 清空搜索」按钮

### 键盘地图

| 键 | 行为 |
|---|---|
| `↑` / `↓` | 列表上/下一项，详情即时跟随（巡检主通道；焦点在输入框时不劫持） |
| `n` / `p` | 树内跳到下/上一处差异，目标行闪烁定位 |
| `1` / `2` / `3` | 树 / JSON / 图片视图 |
| `F` | 仅看不同开关 |
| `D` | 差异高亮开关 |
| `Ctrl+F` | 聚焦搜索框 |
| `Ctrl+B` / `Ctrl+Shift+B` | 收起/展开侧栏 / 差异栏 |
| `Ctrl+O` / `Ctrl+Shift+O` | 选左 / 右目录（单文件模式则为文件） |
| `Ctrl+Enter` | 重新扫描 / 重新比对 |
| `Esc` | 仅当焦点在搜索框：清空并失焦；其余场景无操作（master-detail 下不再有「返回」语义） |

原 `[` / `]` 快捷键废除（由 `↑↓` 接管）；`Esc` 清空槽位的旧行为废除（防误触）。

### 右侧差异栏

- 条目按顶层分组聚类（站点信息 / 线路 1 · B932 / 途经站点 …）
- 条目样式：状态色文字 + 字段名（如「改 · 开往方向」）
- 点击条目 → 树滚动到该行并闪烁一次（`scrollIntoView` + CSS 动画，不触发重渲染）
- 栏底「复制差异清单」按钮：导出多行文本 `开往方向: 福城万达广场 → 福城天虹`（仅左/仅右行为 `下一站: 尚都花园（仅左）`），用于贴到聊天工具报问题
- solo / 一致 / 错误详情时差异栏隐藏（一致时显示「无差异」一行）

### 树交互

- 分组折叠状态在同一对文件内记忆，切换文件对后重置（沿用现行为）
- 「仅看不同」开启时自动展开含差异的折叠组（沿用 `forceOpenIfHasDiff`）
- 嵌套大数组（途经站点、分站列表）默认折叠，标题右侧 `▸ N 项`（沿用）
- 行 hover：行尾出现复制图标，点击复制该行「左值 / 右值」（solo 时单值）
- 组头 hover：复制 JSON 子树按钮（沿用 2026-05-05 设计）

### 最近使用（MRU）

- 目录对、文件对各记 8 条，localStorage 持久化（`recent.dirPairs` / `recent.filePairs`）
- 记录结构：左右路径 + 最后使用时间；扫描/对比**成功**后写入（去重置顶）
- 欢迎页列出：`左 basename ⇄ 右 basename · 相对时间`，hover 显示完整路径，点击 → 填入两侧并自动开始扫描；行尾 × 可删除单条
- 侧栏目录芯片点击弹出的替换浮层中也提供 MRU 快捷项

### 扫描中与取消

- 侧栏进度条 + 「已比对 n / total」（沿用 Channel 节流进度）
- 取消按钮 → `cancel_scan` 命令，生效后静默回欢迎态（MRU 仍在，可一键重来）

### 字体与字号

- 字体沿用：`Inter / PingFang SC / Microsoft YaHei`（UI）+ `JetBrains Mono / Cascadia Code`（字段、值、路径、JSON）
- 树行字号 12px → **13px**，行高 1.6，行 padding 5px —— 舒适度核心调整
- 侧栏 / 差异栏 12px；详情头部 13px；欢迎标题 24px

### 视觉规则

- 差异表达只靠**行底色 + 值文字色**；树行内不再出现徽章（徽章只留侧栏计数、差异栏分类）
- 缺失侧值显示 `—`（`--text-tertiary`）
- 高亮开关（`D`）关闭时树完全素色（差异栏仍可见）
- 沿用：无阴影、无渐变、无 emoji；`null → —`、空串 `(空)`、未识别字段灰标「未识别」等值规则不变（labels.ts 不动）
- 焦点环：`:focus-visible { outline: 2px solid var(--accent); }`

## 4 · 性能专项

### ① 「点浏览选目录」卡顿（主诉求）

**诊断先行**（实施第一步打点量化）：

- 计时 A：点击 → 对话框出现。若大 → 插件在主线程跑对话框消息循环，阻塞 webview 事件循环
- 计时 B：对话框打开期间主窗口是否冻结（拖动窗口测试）

**修复分支**：

- 分支 A（对话框迟出）：新增 Rust 命令 `pick_folder`，在 `spawn_blocking` 工作线程上跑 `rfd::FileDialog::pick_folder`（rfd 自行处理 COM STA 初始化），前端弃用 plugin-dialog 的 `open()`
- 分支 B（OS 枚举慢）：属 Windows 行为（600 张大 PNG 缩略图），无法根治，靠下方绕路通道吸收

**绕路主通道（无论诊断结果，确定做）**：MRU 一点即扫 + 路径粘贴 + 拖放，让日常 90% 场景不碰系统对话框。

### ② 渲染减负

- 单树双值列：DOM 节点数较双 pane 直接减半；同步滚动、占位行逻辑整体删除
- `buildMirrorRows()` 由树与差异栏**各算一遍 → 单处 `useMemo` 共享**
- 展开后的大数组组容器加 `content-visibility: auto` + `contain-intrinsic-size`，离屏行不参与排版绘制
- 侧栏 600 行：行组件 `React.memo` + 固定行高 + 稳定 key；实测仍卡再手写窗口化（不引第三方库；600 量级预期不需要）
- JSON 视图 `format()`（parse + stringify）加 `useMemo`
- 搜索不防抖（600 项子串匹配 < 1ms，防抖反而添延迟感）

### ③ 扫描取消（后端）

- 全局 `AtomicU64` 扫描代号：`scan_directory` 开始时 +1 并记住自己的代号；`cancel_scan` 命令再 +1
- 比对 worker 每处理完一对检查代号是否过期，过期立即停止；`scan_directory` 返回 `Err("cancelled")`
- 前端对 `cancelled` 静默处理（回欢迎态，不弹错误）；现有 `scanSeqRef` 防串台机制保留

### ④ 验收指标（实施完逐条实测，用真实 600 PNG 目录）

| 指标 | 目标 |
|---|---|
| 点「浏览」→ 对话框可交互 | < 500ms，期间主窗口不冻结 |
| 列表搜索每击键渲染 | < 16ms（不掉帧） |
| 打开一对典型差异文件 → 树出现 | < 100ms |
| 切换仅看不同 / 高亮 | < 50ms |
| 取消扫描生效 | < 300ms |
| 扫描全程 UI 可交互 | 回归确认（已达成） |

### 数据流

```
scan_directory ──Channel进度──▶ Sidebar 进度条
      │
      ▼
DirectorySummary.items（轻量 tallies）──筛选×搜索×排序──▶ 列表
      │ 选中（点击 / ↑↓）
      ▼
compare_single / inspect_single ──▶ PairInspection / SideInspection
      │
      ▼
buildMirrorRows()  ←—— 单处 useMemo
      ├──▶ UnifiedTree（树渲染）
      └──▶ DiffRail（差异清单 + 跳转锚点）
```

MRU 在扫描/对比成功后写入。IPC payload 不变（列表轻量、单对按需加载）。

### 错误处理

- 列表错误项：红点 + 错误徽章；点击 → 中央错误卡（错误码 `E_NO_METADATA` / `E_TRUNCATED` / `E_INVALID_JSON` 等 + 文件路径 + 「打开文件」）
- 命令失败（目录不存在等）：顶部横幅，**不清空**上一次结果
- 取消扫描：静默
- 图片加载失败：pane 内可见占位（替换现在的静默 `data-broken` 标记）
